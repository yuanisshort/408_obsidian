# 问题
## 问题1：26考研应该怎么准备？
这个问题经常被问到，首先要在正确的时间做正确的事，这个时间最适合什么？如果你有以下一定的基础，则可以跳过，否则建议按顺序，循序渐进，欲速则不达。

首先408前置内容c语言学了没有？可以看看浙大翁凯的课，跟着写代码，同时有必要进行一些算法的练习（对于没有这方面基础的），可以去刷leetcode，https://leetcode.cn/problemset/，先从简单的模拟题练起（这里的模拟题是指几乎没有算法，而是模拟这个过程的题目），然后可以去代码随想录https://programmercarl.com/，刷除了回溯和动态规划以外的算法题（这部分一般复试笔试或者机试才会考察）。为什么强调这个时候对算法练习？首先算法能力不能一蹴而就，在考研后期学起来不仅性价比低而且吃力。而且这是你初试和复试都必备的一项技能。

其次就是项目的准备，针对于简历项目部分空白的同学，可以先从基础的项目入手，比如最基础的webserver项目，例如通信过程？线程池原理？I/O方式有哪些？多路复用原理？什么是协程？在以上的过程中，你肯定也会需要学习c++的一些知识和特性，例如stl，c++11一些标准，这些见到直接学就完了。

这一部分主要是为了复试，因为初试完之后，完全零基础准备项目，相对来说有点难度，容易被老师问穿。所以只有这个也不太够。

当然webserver这种这是针对简历项目部分完全空白的人，而且这部分说实话，虽然不能说含金量低，但早已经烂大街，对于绝大部分导师而言，他们更在乎的是科研的产出，你自然是要投其所好，例如你可以把热门方向CCF-A或者sci一区的文章复现和修改，哪怕没有开源代码和数据集，也可以去看一下文章内容，了解一下最新的东西，这些往往是老师们关注的，这一部分我打算后面出个项目班。

## 问题2：26考研408应该怎么学？
在同步进行上述问题1的基础上，关于初试408的学习，408这几年也是水涨船高，风格也一直在变，所以我的课程打算每年都会进行针对性的调整和变化。但也有些过程是不可省略的，例如在第一轮学完王道课后，计网和计组可以看湖科大教书匠（**做完所有课后习题（算法题，pv代码题可以看看思路即可，几道即可，因为写这类题太花时间了）*，不要只写选择，那种应用类的大题一定要写，因为这类题难度不高，不适合留在强化，适合基础阶段打基础，如果遇到408真题大题实在写不动，也没关系，可以留在强化阶段，当然遵循原则是能写则写，应写尽写。
