            <div class="content ql-editor"><p><span style="color: rgb(166, 166, 166);">转载自知乎——</span><span style="background-color: rgb(255, 255, 255); color: rgb(166, 166, 166);">千葉原</span></p><p><br></p><h2><span style="background-color: rgb(255, 255, 255); color: rgb(18, 18, 18);">前言</span></h2><p>有同学在备考408过程中发现练习题严重缺乏，其实只要用好真题，题量就完全足够了。</p><p>网上的免费电子版真题有很多，可以说，408是最不需要花钱买资料的科目。</p><p>个人建议做三轮真题。</p><p>第一轮：了解真题考点和解题思路。不限时开卷做。</p><p>第二轮：有基本解题思路，尝试举一反三。不限时闭卷做。</p><p>第三轮：形成知识网络，能够熟练解题。限时闭卷做。</p><p><br></p><h2><span style="background-color: rgb(255, 255, 255); color: rgb(18, 18, 18);">举例</span></h2><p>下面拿一道真题举例。</p><p><br></p><h3><span style="background-color: rgb(255, 255, 255); color: rgb(18, 18, 18);">真题</span></h3><p>2016年第15题</p><p>有如下C语言程序段：</p><div class="ql-code-block-container"><div class="ql-code-block">for (k = 0; k &lt; 1000; k++)</div><div class="ql-code-block">    a[k] = a[k] + 32;</div></div><p>若数组 a 以及变量 k 均为 int 型，int 型数据占 4B，数据 Cache 采用直接映射方式，数据区大小是 1KB，块大小是 16B，该程序段执行前 Cache 为空，则该程序段执行过程中，访问数组 a 的 Cache 的缺失率是：</p><p>A. 1.25%</p><p>B. 2.5%</p><p>C. 12.5%</p><p>D. 25%</p><p>解答：</p><p>数据区大小是 1KB，块大小是 16B，Cache 有 1KB/16B = 64 行，每个块可以装 16B/4B = 4 个数组元素。</p><p>对于数组中任意一个元素 a[k]，只有在 a[k] = a[k] + 32 这行代码中会对 a[k] 进行访问。其中包含读 a[k] 一次，做完加法后写 a[k] 一次。</p><p>每次调入块后加载连续的四个数组元素，a[4i]、a[4i + 1]、a[4i + 2]、a[4i + 3]，其中 i 为整数，i = 0, 1, …, 249。</p><p><img src="https://article-images.zsxq.com/FmRW8PjQCEqVMpS38IyYYliPcYzq"></p><p>缺失率为 1/8 = 12.5%。</p><p><br></p><h3><strong style="background-color: rgb(255, 255, 255); color: rgb(18, 18, 18);">改编一</strong></h3><p>修改代码为：</p><div class="ql-code-block-container"><div class="ql-code-block">for (k = 0; k &lt; 1000; k++)</div><div class="ql-code-block">    a[k]++;</div></div><p>a[k]++ 等价于 a[k] = a[k] + 1，分析过程同上，缺失率为 1/8 = 12.5%。</p><p><br></p><h3><strong style="background-color: rgb(255, 255, 255); color: rgb(18, 18, 18);">改编二</strong></h3><p>修改代码为：</p><div class="ql-code-block-container"><div class="ql-code-block">for (k = 0; k &lt; 1000; k++)</div><div class="ql-code-block">    a[k] = a[k] + a[k];</div></div><p>对于数组中任意一个元素 a[k]，只有在 a[k] = a[k] + a[k] 这行代码中会对 a[k] 进行访问。其中包含读 a[k] 两次，做完加法后写 a[k] 一次。</p><p>每次调入块后加载连续的四个数组元素，a[4i]、a[4i + 1]、a[4i + 2]、a[4i + 3]，其中 i 为整数，i = 0, 1, …, 249。</p><p><img src="https://article-images.zsxq.com/Fm_iIQZpodVZ5E5CjhOj0ABZTruD"></p><p>缺失率为 1/12。</p><p><br></p><h3><strong style="background-color: rgb(255, 255, 255); color: rgb(18, 18, 18);">改编三</strong></h3><p>其实这道题没有充分考察直接映射。</p><p>修改代码为：</p><div class="ql-code-block-container"><div class="ql-code-block">for (k = 0; k &lt; 256; k++)</div><div class="ql-code-block">    a[k] = a[k] + 32;</div><div class="ql-code-block">for (k = 0; k &lt; 256; k++)</div><div class="ql-code-block">    a[k] = a[k] + 32;</div></div><p>修改 for 循环上限为 256，两轮，由于是直接映射，Cache 最多加载 64 个块，即数组中连续 64 × 4 = 256 个数据。</p><p>第一个 for 循环中，每次调入块后加载连续的四个数组元素，a[4i]、a[4i + 1]、a[4i + 2]、a[4i + 3]，其中 i 为整数，i = 0, 1, …, 63。</p><p><img src="https://article-images.zsxq.com/Fp0-V5CKQcam7fq4QTrjklfkub_w"></p><p>第二个 for 循环中，所有需要访问的块都在 Cache 中，不需要调入新的块，考虑某一个块中元素 a[4i]、a[4i + 1]、a[4i + 2]、a[4i + 3]，其中 i 为整数，i = 0, 1, …, 63。</p><p><img src="https://article-images.zsxq.com/Fpg7qiaw_uO-0cMIL_isity3x2f8"></p><p>缺失率为 1/16。</p><p><br></p><h3><strong style="background-color: rgb(255, 255, 255); color: rgb(18, 18, 18);">改编四</strong></h3><p>继续加大难度。</p><p>修改代码为：</p><div class="ql-code-block-container"><div class="ql-code-block">for (k = 0; k &lt; 384; k++)</div><div class="ql-code-block">    a[k] = a[k] + 32;</div><div class="ql-code-block">for (k = 0; k &lt; 384; k++)</div><div class="ql-code-block">    a[k] = a[k] + 32;</div></div><p>修改 for 循环上限为 384，两轮，由于是直接映射，Cache 最多加载 64 个块，即数组中连续 64 × 4 = 256 个数据。</p><p>先考虑第一个for 循环。</p><p>先加载 a[0], a[1], …, a[255]，此时 Cache 已满，下面 a[256], a[257], … a[383] 会替换掉 a[0], a[1], …, a[127]。最后 Cache 中保留 a[256], a[257], … a[383] 和 a[128], a[129], …, a[255]。</p><p>再考虑第二个 for 循环。</p><p>此时 Cache 已满 a[0], a[1], …, a[127] 会重新替换掉 a[256], a[257], … a[383]，a[128], a[129], …, a[255] 仍然保留在 Cache 中，然后 a[256], a[257], … a[383] 会再次替换掉 a[0], a[1], …, a[127]。</p><p>将数据长度分三段。</p><p>第一段 a[0], a[1], …, a[127] ，每个块缺失 2 次，访问 16 次。</p><p>第二段 a[128], a[129], …, a[255] ，每个块缺失 1 次，访问 16 次。</p><p>第三段 a[256], a[257], … a[383] ，每个块缺失 2 次，访问 16 次。</p><p>三段长度相同，缺失率为(2+1+2)/(16+16+16) = 5/48。</p><p>当然，这道题还有很多改编方式，例如将直接映射修改为全相联映射或组相联映射，增加置换算法，这样不断复杂化，我们甚至可以将这道单项选择题改编为一道综合应用题。</p><p><br></p><h2><span style="background-color: rgb(255, 255, 255); color: rgb(18, 18, 18);">后记</span></h2><p>历年真题中每一道真题都是很好的母题，很多新的真题都能看到改编的痕迹。</p><p>采用控制变量法，修改真题中每一个具有考察意义的条件，就能得到几倍于真题的练习题。</p></div>
