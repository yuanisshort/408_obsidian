
# 数据结构-“五步法”计算时间复杂度

#### 一、五个步骤
1. **确定基本操作**：找出代码中影响时间复杂度的关键步骤。这些通常是循环、递归调用或其他重复操作。1. **计算操作次数**：对于每个基本操作，估计其执行次数。这可能是一个常数（如O(1)），也可能与输入大小（如n）有关。1. **确定循环和嵌套关系**：查看循环和递归调用之间的关系。如果有嵌套循环，需要将内部循环的时间复杂度乘以外部循环的迭代次数。1. **汇总和简化**：将各个基本操作的时间复杂度相加，得到总时间复杂度。然后尝试简化表达式，保留最高阶项（例如，去掉常数、较低阶项等）。1. **使用大O表示法**：将简化后的时间复杂度用大O表示法表示，例如O(1)、O(n)、O(n^2)、O(log n)等。
<br>

#### 二、例题

例题1：

**1.确定基本操作：**在这个例子中，基本操作是while循环。

**2.计算操作次数：**在while循环中，i的值每次乘以2，直到i大于n。我们需要计算循环执行的次数。

**3.确定循环和嵌套关系：**这个例子中只有一个循环，没有嵌套关系。

现在我们需要计算循环执行的次数。假设循环执行了k次，那么有：

i*2^k&gt;n

因为i的初始值为1，所以：

2^k &gt; n

为了求出k，我们可以对等式两边取对数：

得到k &gt; log2(n)

所以循环执行的次数k与log2(n)成正比。

**4.汇总和简化：**因为我们只有一个基本操作，所以总的时间复杂度与循环次数成正比，即k。

**5.使用大O表示法：**根据上面的分析，我们可以得出时间复杂度为O(log n)。

所以这个函数的时间复杂度是O(log n)。

例题2：

**1.确定基本操作：**在这个例子中，基本操作是while循环。

**2.计算操作次数：**在while循环中，每次迭代i的值都会增加1，直到i的立方大于n。我们需要计算循环执行的次数。

**3.确定循环和嵌套关系：**这个例子中只有一个循环，没有嵌套关系。

现在我们需要计算循环执行的次数。假设循环执行了k次，那么有：

i^3 &gt; n

因为每次循环i的值增加1，所以循环执行k次后：

k^3 &gt; n

为了求出k，我们可以对等式两边取立方根：

k &gt; n^(1/3)

所以循环执行的次数k与n^(1/3)成正比。

**5.使用大O表示法：**根据上面的分析，我们可以得出时间复杂度为O(n^(1/3))。

例题3：

**1.确定基本操作：**在这个例子中，基本操作是两个嵌套的for循环。

**2.计算操作次数：**外层循环执行n次，内层循环执行2*i次。我们需要计算总的循环执行次数。

**3.确定循环和嵌套关系：**这个例子中有两个嵌套的for循环。为了计算总的循环执行次数，我们需要将内层循环的次数乘以外层循环的次数。

我们可以先计算内层循环执行的总次数。对于每个i，内层循环执行2*i次。所以，对于所有的i，内层循环总共执行的次数为：

2*1 + 2*2 + 2*3 + ... + 2*n = 2*(1 + 2 + 3 + ... + n)

根据等差数列求和公式，我们可以得出：

2*(n*(n + 1)/2) = n*(n + 1)

所以内层循环总共执行n*(n + 1)次。

**4.汇总和简化：**因为我们只有一个基本操作（m++），所以总的时间复杂度与循环次数成正比，即n*(n + 1)。

**5.使用大O表示法：**根据上面的分析，我们可以得出时间复杂度为O(n^2)。这里我们保留最高阶项n^2，忽略了线性项n。

所以这个函数的时间复杂度是O(n^2)。

**3.确定循环和嵌套关系：**这个例子中没有循环，但有递归调用。递归调用的次数与输入n有关。

对于给定的n，函数会递归调用n次，因为每次调用时n都会减1，直到n小于等于1。所以，递归调用的次数与n成正比。

**5.使用大O表示法：**根据上面的分析，我们可以得出时间复杂度为O(n)。

所以这个函数的时间复杂度是O(n)。

例题6：

**1.确定基本操作：**在这个例子中，基本操作是while循环。

**2.计算操作次数：**在while循环中，每次迭代i的值都会增加1，并累加到sum上，直到sum大于等于n。我们需要计算循环执行的次数。

**3.确定循环和嵌套关系：**这个例子中只有一个循环，没有嵌套关系。

现在我们需要计算循环执行的次数。假设循环执行了k次，那么有：

1 + 2 + 3 + ... + k &gt;= n

根据等差数列求和公式，我们可以得出：

k * (k + 1) / 2 &gt;= n

为了求出k，我们可以对等式两边乘以2：

k * (k + 1) &gt;= 2n

这是一个二次不等式，不容易直接求解。但由于k是一个整数，我们可以通过观察发现，当k接近sqrt(2n)时，不等式成立。因此，循环执行的次数k与sqrt(2n)成正比。

**4.汇总和简化：**因为我们只有一个基本操作（sum += ++i），所以总的时间复杂度与循环次数成正比，即k。

**5.使用大O表示法：**根据上面的分析，我们可以得出时间复杂度为O(sqrt(n))。

所以这个函数的时间复杂度是O(sqrt(n))。
